Featured Project — Northwind Analytics Warehouse

Scope: rebuilt the classic Northwind OLTP into an analytics-ready warehouse in BigQuery and built self-initiated executive dashboards in Tableau (the bootcamp did not include the dashboarding).

  • Modeled facts & dimensions; produced an OBT for BI.
  • dbt transformations with Jinja macros, documented contracts, and lineage.
  • Data quality: not-null/unique/relationship tests, source freshness checks.
  • Performance: partitioning, clustering, and incremental models where useful.
  • Docs site + exposures linking the Tableau dashboards to the dbt lineage.
  • Dashboards: Sales Overview, Product Inventory, Customer Overview.

How it’s Built

Warehouse (BigQuery)

  • Star schema with conformed dimensions & surrogate keys; clear naming.
  • Raw ➜ staging ➜ marts/OBT layers with tight separation of concerns.
  • Cost/perf: selective materializations, partitioning & clustering.

Transformations (dbt)

  • Reusable macros & CTE patterns; incremental where appropriate.
  • Tests: not-null, unique, relationships; source freshness; contracts.
  • Docs & lineage; exposures linking models to dashboards.

Visualization (Tableau)

  • Clean KPI strip; interaction patterns (date picker, tooltips, filters).
  • Accessible color/typography; consistent layout system.
  • Executive-ready views for sales, inventory, and customers.
